Culverin

ABOUT

This is a project consistent of creating a Game Engine that allows you to use tools to create video games. It's created for educational purposes, and we are two students that are studying a Video Games Development and Design degree at CITM, Terrassa (UPC).

CORE SUBSYSTEMS

- GameObject/Components Structure

It works like Unity: you can create/delete GameObjects that will be visible in "Hierarchy" window. GameObjects can contain other GameObjects to mantain a child-parent relation. When you select a GameObject, you can see its info in "Inspector" window. You can attach components to it: mesh, material, camera, script, etc.

- Game Mode

when hitting Play Button, all components that have Game functionality will be activated and updated. The game Time is editable through the Configuration Window.

- Frustum Culling for the camera

The cameras of the engine are composed by a frustum. You can activate Culling to ignore all meshes outside the frustum of the main camera. This works well with the next sub-system:

- Quadtree

Editable through configuration window, it will contain static GameObjects of the scene to optimizing the culling when entering Play Mode.

- Mouse Picking & Gizmos

You can select game objects of the scene, that have a Bounding Box, by clicking their geometry. Once selected, you can modify their position, rotation and scale thanks to Gizmos with buttons 1,2,3, or changing the values of the transform from the "Inspector" window

- Resource Manager

You can import different types of files: fbx, textures, etc. and load them into the scene. The resource manager will control which elements are being used to don't overload the same resource.

HIGH LEVEL SUBSYSTEM: SCRIPTING SYSTEM

You can create your own scripts (.cs files), with restricted functionality, and attach them to the GameObjects of the scene.

By the moment, you can only attach one script per GameObject.

All the variables of the script are shown in the inspector (for debuging purposes).

EDITOR OUTPUT VARIABLES: in the inspector, the variables that are recognizable are: INT, FLOAT, STRING, GAMEOBJECT. (other variables will be shown as UNKNOWN TYPE/VALUE).

You can edit variables from the editor to modify the script.

Make sure you don't have compilation errors in your scripts (you won't be able to run the "Game").

You can modify scripts directly from the inspector, by clicking the "Script button". You will see a new window with the code. Once you have modified the script, click "File"->"Save".

Each script has access to the GameObject that it's attached ("GameObject.gameObject").

HOW TO CREATE A NEW SCRIPT:

1)Create-> C# Script.

2)Write the class name to generate the script

WARNING: don't write the same name of an existing script (it won't be generated).

2)Write the class name to generate the script.

Then, your new script is saved into assets folder, and compiled by default.

IMPORT EXTERNAL SCRIPTS:

You can drag and drop a C# script into the engine window to import it.

If it has no errors, it will be compiled successfully. Otherwise you will have to edit it and save changes to recompile it.

EDIT SCRIPTS:

1) To edit it, you have to attach it to a GameObject first: Inspector->Add Component-> Script.

2) You will have to choose between all imported scripts ("Select script...")

3) A new window with the code will apepear. You can edit it and Save changes (File->Save).

SCRIPTS IN THE INSPECTOR:

- Open editor window to edit the script.

- As a component, script has a checkbox to activate/deactivate it when entering Play Mode.

- NAME: name of the script..

- VARIABLES: a list with all the variables of the script (TYPE, NAME , VALUE).
